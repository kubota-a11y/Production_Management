# 社内システムお知らせ(週次)生成スキル

社員向けに「先週、システムの何が変わったか・どう動けばよいか」を伝えるA4資料を作る。
成果物はすべて `docs/weekly-news/` に置く。

## 手順

1. **前号の確認**: `docs/weekly-news/README.md` の発行履歴表を読み、最新号の号数と対象期間の終了日を得る。
2. **履歴の収集**: 対象期間はその翌日〜今日。
   ```
   git log --since="<開始日>" --reverse --date=format:'%m/%d' --pretty=format:'%ad | %s'
   ```
3. **取捨選択と翻訳**(READMEの「資料の書き方ルール」に従う):
   - 社員の稼働に関わる変更のみ採用。経営関連は除外
   - 開発内部の変更(リファクタ・デバッグ・DBカラム・文字コード等)は除外、必要なら「安定性向上」1行に集約
   - 専門用語を日常語へ言い換え、「何が変わった+どう動けばよいか」で書く
   - 顧客名・受注内容は書かない
4. **HTML作成**: 前号の `volXX.html` をコピーして次号 `volXX.html` を作り、以下を更新:
   - Vol番号・発行日・対象期間(ヘッダーとフッター)
   - テーマ別セクション(タグ例: スケジュール/現場作業/受注/その他)。変更が少ない週は1ページでよい
   - 「皆さんへのお願い(今週からの動き方)」ボックス
   - フッターの次号予定日(翌週月曜)
   - ※初回Vol.1のような全体ダイジェストページは2号以降不要
5. **PDF生成**: `docs/weekly-news/` で
   ```
   "/Applications/Google Chrome.app/Contents/MacOS/Google Chrome" --headless --disable-gpu --no-pdf-header-footer --print-to-pdf="社内お知らせ_VolXX.pdf" "file://$PWD/volXX.html"
   ```
6. **検証**: 各 `.page` がはみ出していないか確認する。ヘッドレスChromeで
   `--screenshot --window-size=794,1123` を使い1ページずつPNG化して目視確認(スクリーンショットは
   スクラッチパッドに出力し、リポジトリには入れない)。
7. **記録**: README.mdの発行履歴表に行を追加。
8. **報告**: PNGの確認結果を添えてユーザーに報告。コミット・プッシュは承認を得てから(プロジェクトCLAUDE.mdのルール通り)。
