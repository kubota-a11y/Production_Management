# 社内システムお知らせ(週次)の作り方

毎週月曜日の朝礼・ミーティングで社員に説明するためのA4資料。
Claude Codeで `/weekly-news` を実行すると、前号の発行以降のgit履歴から自動で下書き〜PDF生成まで行う。

## ファイル構成

- `volXX.html` … 各号のA4印刷用HTML(このHTMLを編集して内容を直す)
- `社内お知らせ_VolXX.pdf` … 配布・印刷用PDF
- このREADMEの「発行履歴」… 次号の対象期間の起点になるので、発行のたびに必ず更新する

## PDFの再生成コマンド(このフォルダで実行)

```
"/Applications/Google Chrome.app/Contents/MacOS/Google Chrome" --headless --disable-gpu --no-pdf-header-footer --print-to-pdf="社内お知らせ_VolXX.pdf" "file://$PWD/volXX.html"
```

## 資料の書き方ルール

- 対象は「社員の稼働に関わる変更」のみ。経営戦略・融資・資金計画の話は載せない
- 開発内部の変更(リファクタ・デバッグ・文字コード修正・DBカラム追加など)は載せないか、「安定性向上」として1行にまとめる
- 専門用語を使わない(API・SMTP・DB等は日常語に言い換える)
- 各項目は「何が変わったか」+「社員がどう動けばよいか」をセットで書く
- 顧客名・受注内容などの顧客データは書かない
- 最後に「皆さんへのお願い(今週からの動き方)」ボックスを必ず入れる
