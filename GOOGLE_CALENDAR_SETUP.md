# Googleカレンダー連携 - セットアップガイド

このガイドでは、プリント刺繍加工業向け案件管理システムをGoogleカレンダーと連携させるための設定手順を説明します。

## 📋 前提条件

- Google アカウント（@gmail.com など）
- Google Cloud Console へのアクセス権

## ステップ 1: Google Cloud Project の作成

### 1-1. Google Cloud Console にアクセス

https://console.cloud.google.com にアクセスします。

### 1-2. 新規プロジェクトを作成

1. 上部の「プロジェクト」セレクターをクリック
2. 「新しいプロジェクト」をクリック
3. プロジェクト名に `Production_Management` と入力
4. 「作成」をクリック

### 1-3. 作成したプロジェクトを選択

新規プロジェクトが準備完了するまで待機し、選択します。

---

## ステップ 2: Google Calendar API を有効化

### 2-1. API ライブラリにアクセス

1. 左サイドバーの「API とサービス」→「ライブラリ」をクリック
2. 検索ボックスに `Google Calendar` と入力
3. 「Google Calendar API」をクリック

### 2-2. API を有効化

「有効にする」ボタンをクリックします。

---

## ステップ 3: OAuth 2.0 認証情報の作成

### 3-1. OAuth 同意画面を設定

1. 左サイドバーの「OAuth 同意画面」をクリック
2. ユーザータイプで「外部」を選択
3. 「作成」をクリック

### 3-2. OAuth 同意画面を設定（詳細）

以下の情報を入力：

**アプリケーション情報**
- アプリ名: `Production Management System`
- ユーザーサポートメール: （あなたのメールアドレス）
- デベロッパーの連絡先情報: （あなたのメールアドレス）

「保存して続行」をクリック

### 3-3. スコープを追加

1. 「スコープを追加」をクリック
2. 以下のスコープを検索・追加：
   - `https://www.googleapis.com/auth/calendar`
   - `https://www.googleapis.com/auth/calendar.events`

「保存して続行」をクリック

### 3-4. テストユーザーを追加

1. 「テストユーザー」セクションで「ユーザーを追加」をクリック
2. あなたのメールアドレスを入力
3. 「追加」をクリック

「保存して続行」をクリック

---

## ステップ 4: クライアント ID とシークレットの取得

### 4-1. OAuth 認証情報を作成

1. 左サイドバーの「認証情報」をクリック
2. 「+ 認証情報を作成」をクリック
3. 「OAuth クライアント ID」を選択

### 4-2. アプリケーションタイプを選択

- アプリケーションの種類: `ウェブ アプリケーション`
- 名前: `Production_Management_Client`

### 4-3. リダイレクト URI を設定

以下の URI を「承認済みリダイレクト URI」に追加：

```
http://localhost:3000/api/google/callback
http://192.168.1.8:3000/api/google/callback
```

（LAN IP は実際のサーバー IP に置き換えてください）

### 4-4. 認証情報をダウンロード

1. 作成後、表示されるクライアント ID とクライアント シークレットをメモ
2. または JSON ファイルをダウンロード

---

## ステップ 5: アプリケーション側の設定

### 5-1. 環境変数ファイルを作成

`.env` ファイルをプロジェクトルートに作成：

```env
GOOGLE_CLIENT_ID=<あなたのクライアント ID>
GOOGLE_CLIENT_SECRET=<あなたのクライアント シークレット>
GOOGLE_REDIRECT_URI=http://localhost:3000/api/google/callback
```

### 5-2. .env ファイルを .gitignore に追加

既に `.gitignore` に `.env` が含まれていることを確認：

```
.env
.env.local
```

### 5-3. package.json に依存を追加

```bash
npm install dotenv google-auth-library
```

---

## ステップ 6: アプリケーションの再起動

```bash
npm start
```

## 使い方

### アプリから Googleカレンダーと連携

1. アプリを開く
2. 「📥 コピペ取り込み」タブの上に「📅 Googleカレンダー連携」ボタンが表示される
3. 「連携」をクリック → Google ログイン画面が表示
4. あなたのメールアドレスでログイン
5. アプリへのアクセス許可を確認して「許可」をクリック

### 動作確認

- **アプリ → Googleカレンダー**: 案件を作成すると自動的にGoogleカレンダーに表示
- **Googleカレンダー → アプリ**: Googleカレンダーで案件のイベントを編集すると、アプリに反映
- **削除同期**: Googleカレンダーでイベント削除 → アプリで自動削除

## トラブルシューティング

### エラー: "redirect_uri_mismatch"

**原因**: リダイレクト URI の設定が正しくない

**解決策**:
- GCP の認証情報設定で、ブラウザから実際にアクセスしている URL と一致しているか確認
- ローカルテスト: `http://localhost:3000/api/google/callback`
- LAN テスト: `http://192.168.1.8:3000/api/google/callback`

### エラー: "access_denied"

**原因**: OAuth 同意画面でテストユーザーを追加していない

**解決策**: GCP の OAuth 同意画面でテストユーザーにあなたのメールアドレスを追加

### Googleカレンダーに反映されない

**原因**: API キー/認証情報が設定されていない

**解決策**:
- `.env` ファイルが存在し、正しい認証情報が入力されているか確認
- サーバーを再起動してから試す

---

## セキュリティに関する注意

⚠️ **重要**: `.env` ファイルには認証情報が含まれます。以下を守ってください：

- `.env` は絶対に Git にコミットしない
- `.gitignore` に `.env` が含まれていることを確認
- 本番環境では環境変数を安全に管理してください
